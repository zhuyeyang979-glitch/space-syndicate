# Commercial Playability Gate

本页定义 v0.6 真人首局与 `1280×720` 主桌的最低可玩标准。它保护真实
`scenes/main.tscn` 与 RuntimeGameScreen，不接受静态 demo、组件孤岛或只在高分辨率成立的布局。

真人首局必须进入**真实 RuntimeGameScreen**；Coach 只显示**单主 CTA**；所有运行桌面都必须遵守
**隐藏信息**边界。

本页是目标合同。基线 `main@b2441cc` 仍有常驻竞价、牌架焦点链和旧报价触发语义；
生产代码、场景与测试必须完成迁移后才能把本门槛标记为通过。

## 主桌最小布局

- 在 `1280×720`、`1600×960`、`1920×1080` 下，顶部状态、顶部牌轨、中央星球、
  右侧短详情、底部玩家板、手牌与当前行动必须同时可见。
- 中央星球始终是主视觉。匿名牌轨保持薄轨道，不能挤压地图或遮挡手牌。
- 主桌不为区域牌架、牌序竞价、怪兽战斗下注、反制、合同回应、满手处理或目标选择
  预留常驻大面板。
- 没有临时窗口时，基础焦点顺序为：
  `顶部状态 → 顶部牌轨 → 星球地图 → 右侧详情 → 手牌 → 当前行动`。
- 打开临时窗口后，焦点进入最上层窗口；关闭或结算后恢复到打开前的合法控件。
- 同时只能有一个可操作临时窗口。其他决策只显示短提示，不能形成互相遮挡的模态堆叠。
- 所有临时窗口关闭后必须真正退出布局，不得留下透明 Control、空白占位或不可见焦点节点。

## 区域牌架

- 区域牌架默认关闭，不占主桌空间。
- 单击区域只选择并旋转星球，不自动打开牌架。
- 双击区域、点击“查看牌架”或使用对应键盘/手柄确认操作后才打开区域牌架。
- 打开、关闭、悬停、滚动和重新打开牌架不得刷新挂牌。
- 玩家可以预览暂时买不起或暂时无法打出的牌，并看到简短合法条件。
- 公开预览不创建报价。玩家明确选择购买某张挂牌时，才请求权威报价规则锁定
  5 秒 `world_effective` 报价。
- 关闭窗口不延长报价。再次打开时，有效报价显示剩余时间；失效报价必须重新请求。
- 购买成功后默认关闭牌架；只有玩家启用“连续浏览”本地设置时才保持打开。
- Back、Esc 与手柄取消键关闭最上层可关闭窗口，不得在其后同时打开暂停菜单。

## 牌序竞价与强制窗口

- 完整“牌序竞价”窗口只在共享出牌窗口处于 `public_bid` 阶段时出现。
- `planning`、`lock`、`resolve` 与 `idle` 阶段只允许顶部牌轨显示简短阶段或结果，
  不得保留完整竞价面板。
- 怪兽战斗下注只在真实战斗下注时机出现，继续执行强制选择、15 秒、全局冻结、
  全员完成可提前结束和超时默认选择。
- 反制、合同回应、满手处理、怪兽目标和玩家目标窗口只在对应权威决策存在时出现。
- `ForcedDecisionRuntimeScheduler` 统一决定当前可操作窗口；强制决策不能通过 Back
  打开暂停菜单绕过。
- 窗口结束后自动关闭、释放焦点并恢复此前合法焦点。
- 非当前决策玩家只看到短提示，不得看到对方的私人选择内容。

## 商路按需查看

- 新局默认“商路已隐藏”，`selected_trade_product_id=""`。
- 普通全图视角也不得自动显示商品流线。
- 玩家点击“查看商路”并选择一种具体商品后，地图才显示该商品当前或最近短观察窗内
  的真实流向。
- 显示内容只来自实际成交、仓库转运或区域基础消费的一跳公开摘要；
  不显示潜在候选路线。
- 关闭或切换商品只改变本地显示，不重新计算物流，不改变现金、GDP、仓库、需求积压
  或 AI 行为。
- 路线显示偏好不进入共享权威存档；新局始终默认关闭。

## 首局动作链

首局 Coach 至少要能带玩家完成：

1. 选择推荐区域；
2. 主动打开区域牌架；
3. 购买第一张区域牌；
4. 打出第一张发展牌并建立首个经济项目；
5. 打开经济总览；
6. 购买并打出第二张当前公开挂牌；
7. 查看顶部牌轨和公开线索；
8. 阅读怪兽压力；
9. 选择一种发展路线；
10. 在经济建立前或建立后，自愿召唤持有的起始怪兽。

起始怪兽召唤不是设施、购牌或经济发展的前置条件。首局路线总结完成前，怪兽战斗下注等
全局冻结窗口必须延后，不能抢走基础教学焦点。
Coach 的操作始终绑定本地真人席位；查看 AI 公开档案不能把购买、出牌或开窗操作打到 AI 身上。

## 玩家可读性与隐私

- 经济总览先显示 KPI，再给“扩 GDP / 护商路 / 压竞争”三条短建议。
- 市场使用“市场正常需求 / 市场待满足需求 / 市场追赶消费”。
- 工厂使用“已售出 / 区域基础消费 / 已入库 / 浪费产能”。
- 路线按钮使用“查看商路 / 隐藏商路”，默认状态使用“商路已隐藏”。
- 不显示 `backpressure`、`milliunits`、`sink`、`route candidate`、原始状态枚举、
  owner/controller/world bridge 或 AI 内部评分。
- 不泄漏对手现金、对手手牌、私人商品供应者身份、AI 计划、真实隐藏业主或私人报价绑定。

## 自动验收

完成实现后，至少运行：

```powershell
pwsh -File tools/invoke_commercial_playability_gate.ps1
godot --headless --path . --script res://tests/transient_gameplay_windows_v06_test.gd
godot --headless --path . --script res://tests/route_visibility_opt_in_v06_test.gd
godot --headless --path . --script res://tests/layout_scene_smoke_test.gd
godot --headless --path . --script res://tests/smoke_test.gd --check-only
```

`invoke_commercial_playability_gate.ps1` 在独立 `APPDATA/LOCALAPPDATA` 中逐阶段运行门禁，
默认每阶段最多 20 秒、整套最多 90 秒。它会把首个超时阶段、阶段耗时、Godot 退出码和
`SCRIPT ERROR` 写入 `.codex-godot/commercial-gate-runs/`。该分段执行不删减断言，只避免
一个无输出的长进程掩盖首个阻塞点。

可用 `-Stage` 单独复验：

```powershell
pwsh -File tools/invoke_commercial_playability_gate.ps1 -Stage action_chain
```

阶段包括：

- `documentation`
- `layout_1280`
- `layout_1600`
- `layout_1920`
- `cta_open_rack`
- `cta_buy_recovery`
- `optional_summon`
- `action_chain`

完整发布验收还必须运行完整 `tests/smoke_test.gd`，并检查 `1280×720` 有头截图：

- 无临时窗口时，主桌没有牌架、竞价或下注常驻占位；
- 区域牌架打开时不遮挡主要购买按钮；
- 强制窗口出现时不存在第二个可操作模态；
- 商路关闭时地图没有商品流线；
- 商路打开时只显示所选商品的真实流向；
- 窗口结束后焦点恢复且布局完全释放。
