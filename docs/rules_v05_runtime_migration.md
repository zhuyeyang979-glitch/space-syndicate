# Ruleset v0.5 运行时迁移与引用合同

## 当前状态

v0.5 设计规则已经确认；当前可运行版本仍是 v0.4。本文件只规定迁移顺序、所有权和验收门，不声称 v0.5 已经在 Godot 运行时实现。Git 拓扑、CurrencyAmount、clock domains 和跨领域 receipt 的先决合同见 `docs/rules_v05_migration_foundation_contract.md`。

生产代码只有在全部 v0.5 conformance gate 通过后才能把活动规则集切到 v0.5。迁移期间不得维护两个同时生效的规则引擎，也不得把 v0.5 文档数字零散硬编码进展示层。

## 已确认的数字参数

| 参数 | v0.5 值 |
| --- | --- |
| 区域控制 | 个人归属 GDP 占比至少 3000bp，且唯一最高 |
| 深度 I–VI | 3/90、4/130、5/180、6/230、7/290、8/360 |
| 资格保持 | 10 个有效游戏秒 |
| 终局审计 | 120 个有效游戏秒 |
| 审计失败冷却 | 30 个有效游戏秒 |
| 产业产能档位 | 15、40、80、140 GDP/min 对应 1–4 |
| 卡牌组 | 8 秒，6 秒组织 + 2 秒锁牌，标准每席 2 张 |
| 优先出价 | 0、50、100，进入下场怪兽公共池 |
| 购买窗口 | 12 秒，所在区 80%，相邻区 100% |
| 怪兽预告 | 5 秒 |
| 怪兽赌局 | 8 秒下注；底注 5%–10%；绝对加注最小单位 50；次数无上限 |
| 怪兽战斗上限 | 快速 30、标准 45、专家 60 秒 |
| 合约回应 | 8 秒 |
| 情报竞猜 | 押 100，猜对净赚 100，私密跟踪 60 秒 |
| 天气 | 预报 90 秒，持续 90 秒，同区不叠加 |
| 财务危机 | 20 秒 |

标准怪兽底注率的设计默认由同一纯函数产生：`clamp(5 + participant_count - 2 + highest_rank_bonus, 5, 10)`，其中 I–IV 的 `highest_rank_bonus` 为 0–3。用户直接确认的是 5%–10% 边界；该危险度公式是 v0.5 标准模式默认值。底注使用开窗瞬间的可用现金快照并保留到 0.01，同一战斗只扣一次。普通加注只允许在该 8 秒下注窗口内发生，每笔为 50 的正整数倍并带幂等交易 ID；全押是唯一尾数例外。该 8 秒使用不受世界暂停影响的 UI 实时时钟。窗口关闭后允许卡牌和升级干预，但不得换边、追入或重开赌盘。

5. 实现 10 秒资格、120 秒审计、审计名单、30 秒冷却和终点排序。
6. 单独完成怪兽整场赌局纵向切片，包括暂停、托管、加注、战斗、派奖、破产和存档。
7. 完成区域复兴、财务危机、天气和军队收束。

不能先做胜利或产业颜色，再继续使用“按项目等级瓜分整城 GDP”的旧结果；胜利、产能、合约和情报必须共享同一份玩家实际归属 GDP。

## 单一所有权

### GDP 与项目

`GdpFormulaRuntimeController` 只生成结构化 GDP 行。城市项目状态只把这些行守恒分配到具体项目和玩家份额。生产、需求、通商、合约不得各自复制一套份额公式。

区域固定为生产 2、需求 2、通商 1，项目最高 IV。项目最高份额并列时 `controller_player_index=-1`。删除活跃城市最低 GDP 保底；被毁区域和总 GDP 为 0 的区域全部占比为 0。

### 胜利与审计

建立单一 `VictoryControlRuntimeController`，拥有区域控制、Top N、10 秒保持、120 秒审计、审计名单、30 秒冷却、终点排序和星球毁灭结算。合格领先比较器固定为 Top N GDP、控制区总数、现金总账；同值并列领先者同时入名单。终点候选只来自审计名单内仍合格者，同一结算更新中新升至领先者必须先入名单并公开再排名。UI 和 AI 只消费 public/private snapshot，不复制判断。

审计公共快照只为名单玩家公开准确现金总账、项目份额、个人归属 GDP、合约、仓储和金融头寸；现金总账等于可用现金加托管现金。不得泄露手牌内容、私人情报、隐藏怪兽对应关系或 AI 内部评分。

### 产业产能与卡牌组

`CardPlayEligibilityRuntimeService` 计算单牌的产业、具名商品和影响力合法性。`CardResolutionQueueRuntimeService` 拥有同组累计占用、提交锁定、结算释放和存档 receipt。展示层不得自行聚合 GDP。

支持无色、单色、双色、二选一和具名商品；每张牌最多两个主要合法性条件。被反制牌正常消耗；系统漂移无目标时通过库存事务退牌并冷却 3 秒，不能直接向手牌数组塞回对象。

### 怪兽赌局

`MonsterRuntimeController` 唯一拥有 `battle_id`、参战快照、底注现金快照、托管资金、原子加注记录、累计实际伤害、战斗时限和派奖。状态至少分为 betting、battle、settled。

只有 betting 阶段冻结星球；betting 使用 unscaled UI timer，不能因世界暂停而停止自己的 8 秒倒计时。下注方向在首次支付底注时锁定；观望者不能战中追入，后到怪兽不加入，升级满血不清累计伤害。总池等于 Queue 唯一产出的 `public_wager_pool_receipt` 加全部底注和加注，获胜方按最终下注比例分配；平局退玩家资金、公共池滚存，无人押中则整池滚存。Monster 不得重新读取或计算卡牌优先出价。

### 合约、情报、复兴和财务危机

合约端点必须保存真实 `source_project_id`、`target_project_id` 和同一 `product_id`，收益回到具体 GDP 行。普通超时中性拒绝；每项目一份主合约。暂时受损只设置 `delivery_blocked/effect_suspended`，交付可以为零但合约时间继续；修复同一 ID 后可恢复交付。区域毁灭会 tombstone 旧项目和旧合约，新建同商品项目不得续接旧约。

情报服务保存每玩家每项目一次锁定竞猜、100 押金、私密结果、60 秒跟踪和最后已知快照。公共 Dossier 只展示当前查看者获准看到的数据，不拥有真相状态。

区域复兴是 destroyed 到 damaged/buildable 的独立状态事务，不是普通 repair。I–IV 分别恢复 25%/40%/60%/80% 基础耐久及牌面列出的项目位和路线端点；旧 project_id、旧份额、旧合约和旧 GDP 永久失效。全星球毁灭判定只等待判定前已经锁定并进入结算队列的复兴，手牌不延迟毁灭。

现金归零进入可保存的 20 秒危机状态。危机期间限制新承诺，收入继续；紧急售牌每次危机最多一次，按规则定义的购买价基准的 50%。合并升级实体的成本基准尚未由玩家规则明确，作为 SS05-10 blocker 登记，不得从 catalog 当前价格推断。赌局托管资金在结算前不触发危机，结算后再检查现金。FinancialDistress 只发出幂等 `elimination_requested`，跨领域 mutation 由 EndStateSettlement 编排。

## 关键实现差距

- 当前 Ruleset Bridge、主协调器及多项核心服务仍只接受 `v0.4`，不能只改资源文件名。
- 当前胜利仍是现金目标和较短倒计时，必须整体替换为 GDP 控制审计。
- 当前 GDP 和城市项目仍有旧的整城分配、城市拥有者或最低 GDP 回退路径。
- 当前卡牌只具备旧区域份额门槛，没有六类产业产能和同组占用。
- 当前怪兽赌局强制参与、使用百分比追加、过早结算且按人数平分，与 v0.5 不兼容。
- 当前现金归零立即淘汰，被毁区域没有独立复兴入口。
- 当前情报仍包含城市或卡牌归属竞猜，合约仍保留区域／城市端点语义。

这些差距必须作为替换项迁移，不得以兼容分支长期并存。

## v0.5 Conformance Gate

### GDP 与胜利

- 29.99% 不控制；30.00% 且唯一最高才控制；30% 并列双方都不控制。
- 被停用项目的 GDP 贡献立即归 0；只有区域被毁或区域总 GDP 为 0 时，所有人的区域控制占比归 0。
- Top N 只计算玩家实际归属 GDP；深度 I–VI 边界值全部覆盖。
- 达标 9.99 秒不触发，10 秒触发；保持期间跌破会重置。
- 120 秒中途不取消，终点重新验证；无人达标时进入 30 秒冷却。
- 终点只评审计名单内仍合格者，依次按 Top N GDP、控制区总数、现金总账和并列规则排序；同刻新领先者先入名单。

### 隐私

- 审计名单玩家的准确经济资产对所有席位公开，未入名单玩家仍私密。
- 加入名单后本次审计内不重新隐藏；保存加载后名单与剩余时间一致。
- 公共快照永不包含具体手牌、私人情报、未揭示怪兽归属或 AI 内部评分。

### 产业与卡牌

- 每种商品恰好一个产业类别；14/15、39/40、79/80、139/140 边界正确。
- 同组两张牌累计占用；双色、二选一和具名商品要求正确。
- 提交后 GDP 变化不追溯取消；组完成只释放一次，保存加载不重复释放。
- 8 秒严格为 6+2；教学 1 张、标准 2 张；全席确认可提前封盘。

### 怪兽赌局

- 底注率始终为 5%–10%，现金只在开窗时快照，底注只扣一次。
- 同一玩家连续 1、10、100 次绝对加注正确累计，无人为次数上限。
- 0、负数、普通加注非 50 倍数、超现金、窗口关闭、换边和重复交易 ID 都原子拒绝；全押尾数是唯一例外。
- 加注不重新计算百分比；观望者不能追入；后到怪兽不加入。
- 治疗和满血升级不减少累计伤害；击倒、超时、平局、无人押中和比例派奖资金守恒。
- 只有 8 秒 betting 阶段冻结，battle 阶段卡牌和技能正常可用。

### 合约、情报、复兴与危机

- 合约严格使用真实项目 ID 和同商品；控制并列无回应权，普通超时无处罚。
- 每项目第二份主合约被拒绝；项目毁灭不会留下幽灵 GDP。
- 猜中、猜错和 60 秒份额跟踪只进入当前玩家私密快照。
- 摧毁立即归零；复兴不恢复旧项目、旧份额、旧合约或 GDP。
- 现金归零先进入 20 秒危机，收入可救回；超时才淘汰。
- 全星球毁灭时取消未结算赌局、返还玩家托管资金，再中止普通审计并按准确现金总账结算，同额并列。

## 完成定义

v0.5 只有在规则 Resource、存档升级、AI、教程、玩家文案、公共隐私快照和上述门禁全部通过后才可宣称“已实现”。切换同一提交中应把生产桥指向 v0.5，并删除所有会让现金胜利、城市拥有者、卡牌归属竞猜和旧百分比追加赌局重新生效的生产回退路径。
