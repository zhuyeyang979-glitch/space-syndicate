# Hearthstone-grade UX Matrix

This matrix defines the commercial vertical-slice target for Space Syndicate without copying Blizzard IP, assets, icons, text, or card rules. The benchmark is product structure: immediate readability, strong feedback, and a complete play loop.

| Module | Local component | Target behavior | Current gap | v1 completion | Screenshot / frame gate |
| --- | --- | --- | --- | --- | --- |
| board staging / 桌面舞台 | `VerticalSliceShowcase`, `GameScreen`, `PlanetBoard` | Player reads public track, central planet table, inspector, resources, hand, and action area in one scan. | Main game is playable but showcase entry is not isolated for a 45-90 second demo. | Add a deterministic showcase table with public track, planet stage, inspector, hand rack, resource strip, bid strip, and overlay layer. | `showcase_board_idle_1600x960.png` |
| hand cards / 手牌 | `HandRack`, `CardFace`, `CardUI` | Cards feel like objects that can be lifted, selected, dragged, and returned. | HandRack v3 has the base feel; it needs a showcase scene that proves it in context. | Use HandRack v3 in the showcase with hover, selected, valid drag, invalid drag, and stable right-inspector detail. | `showcase_card_hover_1600x960.png`, `showcase_card_drag_valid_1600x960.png`, `showcase_card_drag_invalid_1600x960.png` |
| card hover / 卡牌悬停 | `HandRack`, `VisualEventLayer` | Hover raises card, pushes neighbors, adds glow, and previews the card without changing stable selection. | Hover is implemented but not captured in a commercial sequence. | Add `card_hover_glow` visual event and a showcase hover frame. | `showcase_card_hover_1600x960.png` |
| card drag / 卡牌拖拽 | `HandRack`, `TargetingOverlay` | Drag separates card from rack, follows intent, shows valid/invalid target feedback, and never runs rules directly. | Drag feedback exists but needs target arrow and short map reason copy. | Add `TargetingOverlay` states for valid, invalid, no-map, and no-target card release. | `showcase_card_drag_valid_1600x960.png`, `showcase_card_drag_invalid_1600x960.png` |
| target selection / 目标选择 | `TargetingOverlay`, `RightInspector` | Map shows a short drop reason; inspector carries full explanation. | Existing drag label is local to game screen. | Add reusable target arrow and reason contract for the showcase and future scenario fixtures. | `showcase_card_drag_valid_1600x960.png` |
| card play / 出牌演出 | `VisualEventLayer`, `ShowcaseDirector` | Card flies from hand to target/public track, target flashes, and resources float. | Existing play action is rules-first; showcase needs UI-only choreography. | Add `card_play_flyout`, `target_arrow`, `cash_gain_float`, and `gdp_delta_float`. | `showcase_card_play_frame_00.png`, `showcase_card_play_frame_08.png`, `showcase_card_play_frame_16.png` |
| summon / 召唤 | `MonsterEventPresenter`, `VisualEventLayer` | Monster spawn is a region pulse plus token drop and threat color. | Monster rules exist, but display language is not isolated. | Add `monster_spawn_pulse` event and presenter factory. | `showcase_monster_spawn_1600x960.png` |
| attack / 攻击 | `MonsterEventPresenter`, `CombatEventPresenter`, `VisualEventLayer` | Monster attack has windup, impact, city flash, and readable source/target. | Full rules are not needed; v1 needs readable animation frames. | Add `monster_attack_windup` and `monster_attack_impact` frames. | `showcase_monster_attack_frame_00.png`, `showcase_monster_attack_frame_12.png`, `showcase_monster_attack_frame_24.png` |
| damage / 受损 | `CityDamagePresenter`, `RouteEventPresenter`, `VisualEventLayer` | City/route damage shows crack/red edge, HP/GDP float, and a one-line reason. | Damage is visible in data but not packaged as a vertical-slice moment. | Add `city_damage_crack`, `route_damage_spark`, `gdp_delta_float`. | `showcase_city_damage_1600x960.png` |
| death/cleanup / 退场 | `VisualEventQueue`, `VisualEventLayer` | Finished events leave no stale UI and respect object pooling/cleanup limits. | Existing overlays are transient but not event-queue based. | Queue caps same-screen events at 32 and expires/clears snapshots. | `visual_event_smoke_test.gd` |
| resource feedback / 资源变化 | `VisualEventLayer`, `AudioEventBus` | Cash/GDP changes float near source and emit silent audio hooks. | Resource changes are often text updates only. | Add `cash_gain_float`, `gdp_delta_float`, and corresponding audio hooks. | `showcase_card_play_frame_16.png`, `showcase_city_damage_1600x960.png` |
| turn/phase feedback / 阶段反馈 | `ShowcaseDirector`, public track strip | Sequence time and scenario phase are legible at a glance. | Scenario Lab will own real flow; v1 needs a deterministic local timeline. | Add `hearthstone_grade_sequence.json` and director-controlled phase labels. | `showcase_board_idle_1600x960.png` |
| log/replay / 日志复盘 | `VerticalSliceShowcase`, `RightInspector` | Each spectacle moment also has one human-readable explanation. | Existing logs are broad; showcase needs short explanation copy. | Add inspector text per showcase stage and no private-info leakage. | all showcase frames |
| audio hooks / 音效钩子 | `AudioEventBus`, `AudioEventRegistry` | Hooks exist even when default output is silent. | No dedicated showcase audio event surface. | Add registry for hover/click/card/monster/city/final-countdown events. | `visual_event_smoke_test.gd` |

Reduced motion is mandatory for this slice. In reduced motion mode, the same semantic events render as static highlights and labels without flyout trails or pulse spam.
