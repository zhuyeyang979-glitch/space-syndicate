# 区域牌架全面随机化契约 v0.6

## 1. 范围与权威

本文定义每个区域可付费购买的非商品普通牌牌架。顶部免费商品履带、商品履带视野和商品领取顺序保持独立，不属于本契约。

玩家规则权威仍为 [`tabletop_rulebook_v06.md`](tabletop_rulebook_v06.md)。本文补充实现必须遵守的数据字段、owner 边界、确定性、存档、隐私和测试要求。

现役区域牌架 / CardFlow / DistrictPurchase owner 是区域挂牌与购买的唯一权威，独占：

- 当前区域牌位。
- 合法区域牌池快照。
- 确定性 shuffle bag。
- gameplay RNG 的区域牌架域状态。
- 全局唯一牌状态。
- 刷新序号。
- 5 秒购买报价。
- 购买、发牌与单槽补充的 exact-once transaction lineage。

其他系统边界：

| 系统 | 可以做什么 | 不可以做什么 |
| --- | --- | --- |
| `RegionInfrastructureRuntimeController` | 提供区域、设施槽、产权、等级和潜在目标事实 | 抽牌、拥有牌袋、决定下一张类别 |
| `ProductMarketRuntimeController` | 管理商品价格、趋势和金融市场状态 | 因“市场设施牌”挂牌而拥有或修改区域牌架 |
| `CommodityFlowRuntimeController` | 在市场设施建成并安装具体商品需求后拥有积压与物流经济状态 | 反向改变牌袋或挂牌顺序 |
| `ForcedDecisionRuntimeScheduler` | 决定强制临时决策的优先级与阻塞语义 | 拥有购牌报价、出价、现金或牌位 |
| `OverlayLayer` | 呈现区域牌架、报价与临时窗口 | 拥有随机、牌袋、报价计时、现金或卡牌 |
| AI | 读取公开牌位并提交购买/出牌意图 | 查看未来牌袋、RNG 状态或私有玩家状态 |
| WorldBridge | 提供纯数据世界事实并原子应用权威结果 | 过滤类别、抽牌、重洗或按 UI 状态刷新 |
| `main.gd` | 转发仍未迁出的窄接口 | 新增牌架 owner、兼容阶段或影子状态 |

## 2. 玩家可见规则

每个区域的全部挂牌槽位都来自同一个合法非商品普通牌池。只要当前模式启用，工厂、市场、道路、码头、空港、仓库、城市发展与策略、怪兽、军队、组织、互动和反制牌都能出现在任意槽位。

合法开局包括：

- 市场牌先于工厂牌出现。
- 工厂牌先于市场牌出现。
- 某区域暂时只有交通、仓库、战术或单位牌。
- 不同区域得到完全不同的牌型组合。

区域牌架不承诺类别顺序。玩家当前现金不足、资产不足、尚未拥有设施或暂时没有可立即打出的目标时，牌仍可以挂牌；UI 显示购买条件与出牌条件，不承诺当前立即可用。

## 3. 数据字段

每个可进入区域牌架的卡牌资源至少提供或可规范化得到：

| 字段 | 规则 |
| --- | --- |
| `card_id` | 稳定、全局唯一、跨存档不变化 |
| `region_supply_family_id` | 稳定家族 id，用于默认家族等权 |
| `region_supply_weight` | 非负整数权重；默认 `1`，`0` 等同不进入当前模式 |
| `mode_enable_tags` | 当前模式启用条件 |
| `retired` | 为 `true` 时不得进入现役牌池 |
| `unique_scope` | `none`、本局全局唯一或现役数据允许的其他明确唯一范围 |
| `target_profile` | 用于判断本局是否存在潜在合法目标，不读取某位玩家瞬时资源 |
| `resource_revision` | 卡牌资源与规则字段修订，用于存档/池版本校验 |

权重只用于随机抽取，不向玩家显示。默认所有合法家族权重相同；工厂和市场不得拥有隐藏的先后权重。若同一家族有多个可挂牌资源，其内部选择也必须使用稳定、可审计的数据规则，不能按代码分支暗中优先某个等级或类别。

实现可以使用不同字段名，但公开契约、存档迁移和测试必须能一一映射到上述语义。不能通过中文卡名或资源路径片段推断类别、权重或目标。

## 4. 合法池构建

每个区域按稳定 `region_id` 构建合法池。只允许以下硬过滤：

1. 当前模式启用。
2. 未退役。
3. 卡牌资源可加载，必需字段合法。
4. 全局唯一牌未违反唯一状态。
5. 本局至少存在一个潜在合法目标。

“潜在合法目标”按地图拓扑、模式能力、席位结构和目标类型判断。例如本局存在可建设市场的存续区域，就可以保留市场牌；不要求当前玩家已经拥有工厂、现金、资产或商品需求。只有在本局规则内永久不可能合法结算的牌才过滤。

以下信息不得参与合法池构建：

- 当前玩家现金、资产、手牌或 GDP。
- 当前设施产权、短暂占用或临时损伤，只要未来仍可能出现合法目标。
- 当前是否存在配套工厂、市场、路线或仓库。
- 当前镜头、悬停、滚动、焦点、窗口开关或控制器输入。
- AI 计划、评分或私有路线。
- 系统时间、帧数和非 gameplay 随机源。

合法池输入必须先规范化排序。区域按稳定 `region_id`，槽位按稳定 `slot_id`，家族按 `region_supply_family_id`，卡牌按 `card_id` 排序；随机只能发生在规范化之后。

## 5. 确定性 shuffle bag

### 5.1 RNG 域

区域牌架使用权威 gameplay seed 的独立确定性域，例如由以下稳定输入派生：

```text
region_supply_rng_seed
= stable_hash(
    gameplay_seed,
    "region_supply_v06",
    region_id
  )
```

`stable_hash` 必须是跨平台、跨进程稳定的明确算法，不能使用语言运行时可能变化的默认对象哈希。派生域不得消耗战斗、商品履带、天气或 AI 的随机序列。

### 5.2 牌袋规则

每个区域维护自己的无放回牌袋：

1. 从规范化合法池生成家族票据；`region_supply_weight` 决定显式票据权重。
2. 使用该区域 gameplay RNG 域确定家族顺序。
3. 家族内部从规范化资源子袋无放回抽取。
4. 抽取结果直接进入目标槽位，不再按类别重排。
5. 牌袋耗尽时提高 `bag_epoch`，再以同一合法池和连续 RNG 状态确定性重建。

允许使用语义等价的确定性加权无放回算法，但必须满足：

- 相同种子、合法池、唯一牌状态和购买序列得到相同结果。
- 不同种子在统计上能够产生不同结果。
- 权重可从数据与测试审计。
- 没有工厂阶段、市场阶段、保证槽或依据上一张类别选择下一张类别的分支。

### 5.3 全局唯一牌

唯一状态至少区分：

- `available`
- `listed(region_id, slot_id)`
- `purchased`
- `consumed_or_retired`

唯一牌进入某个公开槽位时即从其他区域后续抽取中排除。购买、移除或销毁后的状态转换由同一权威事务完成；多次应用同一 transaction id 不能让唯一牌重复出现。

## 6. 初始化、补槽与报价

新局初始化顺序固定：

1. 规范化模式与卡牌资源。
2. 建立全局唯一状态。
3. 按稳定 `region_id` 建立区域牌袋。
4. 按稳定 `slot_id` 填充每个区域的模式定义槽位。
5. 提交首个公开牌架 revision。

购买生命周期：

1. 玩家打开区域牌架只读取公开牌位，不抽牌。
2. 公开预览不创建报价。
3. 玩家明确选中或确认挂牌后，权威 owner 建立 5 秒 `world_effective` 报价。
4. 购买提交原子验证挂牌身份、报价、日照、价格、手牌上限和其他现役购买条件。
5. 成功后只清空被购买槽位、扣款、发牌并为该槽抽取一张补牌。
6. 提高该区域 `refresh_sequence` 与公开 revision。
7. 默认关闭购牌窗口；“连续浏览”仅影响本地呈现。

失败、取消、报价过期或窗口关闭不补槽。其他槽位的 `card_id`、报价和刷新 lineage 不变化。

## 7. UI 生命周期

区域牌架默认关闭且不占主桌布局空间。单击区域只选择区域；双击区域、“查看牌架”或对应键盘/手柄操作才打开。

以下行为不得触发抽牌、补槽、重洗或 revision 变化：

- 打开、关闭或重新打开。
- 悬停、滚动、翻页或切换焦点。
- 改变地图缩放、观察方向或商路显隐。
- 切换鼠标、键盘与手柄。
- 报价仍有效时重新打开窗口。

Back、Esc 和手柄取消关闭最上层可关闭窗口并恢复此前合法焦点。窗口关闭不延长报价；再次打开时显示权威剩余时间。Presentation 节点不能保存“下一张牌”或本地补槽队列。

## 8. 存档与恢复

存档至少包含：

```text
region_supply_schema_revision
region_slots[region_id][slot_id] -> card_id
region_bags[region_id] -> {
  family_order_or_equivalent_state,
  family_draw_index,
  family_subbags,
  bag_epoch,
  rng_state
}
global_unique_card_states
refresh_sequence_by_region
public_revision
purchase_and_refill_transaction_lineage
```

若 RNG 算法允许从完整状态重现剩余顺序，可以保存算法 id、状态和游标；否则直接保存剩余牌袋顺序。两种方式都必须让读档后的下一张牌与未读档对局完全一致。

读档是恢复，不是新局初始化：

- 不重建已有牌位。
- 不重新洗牌。
- 不自动填充已经有牌的槽位。
- 不重复应用最后一次购买或补槽。
- 不改变唯一牌位置。
- 不因 UI 首次打开而生成 revision。

存档迁移必须有明确 schema 版本、单次迁移标记和确定性输入。旧的保证槽、类别阶段或下一类字段不得继续成为现役抽牌分支；迁移后只保存统一牌位与牌袋状态。

## 9. 公共快照与隐私

公开牌架快照可以包含：

- `region_id`
- `slot_id`
- 当前 `card_id`
- 公开基础价与购买条件
- 公开日照资格
- 公开报价剩余时间，仅对依法可见的报价主体
- 公共 `refresh_sequence` / revision

快照不得包含：

- 剩余牌袋顺序。
- RNG seed、RNG state、抽取游标或 `bag_epoch` 推导细节。
- 未挂牌唯一牌的位置。
- AI 评分、未来购买计划或路线计划。
- 其他玩家现金、资产、手牌或私人报价意图。

AI 使用与玩家相同的公开牌架投影。测试可以通过专用权威接口检查牌袋，但该接口不能进入生产 UI 或 AI 输入。

## 10. 与其他本轮规则的集成

- 市场设施牌只是区域公共设施牌；它不属于 `ProductMarketRuntimeController` 的商品金融市场。
- 市场牌可以先出现并建成。具体商品需求安装完成后，市场积压由 `CommodityFlowRuntimeController` 独占，与牌架是否打开无关。
- 商品路线默认隐藏。商路显隐不消耗区域牌架 RNG，也不改变合法池或 AI。
- 区域购牌是主动临时窗口；牌序竞价和怪兽下注是时机触发的强制临时窗口。窗口呈现不拥有卡牌或经济状态。
- 购买市场、工厂、仓库或交通牌后，对设施和经济的影响由各现役 owner 在出牌结算时处理；牌架 owner 只负责挂牌、报价、购买和发牌。

## 11. 必须删除的现役分支

生产路径和 focused tests 不得保留：

- 工厂优先、市场后置或购买工厂后解锁市场。
- 工厂、市场、城市发展或怪兽固定保证槽。
- 依据上一张挂牌类别选择下一张类别。
- 打开/关闭 UI 触发刷新。
- 读档后重新整理牌架。
- AI 或 UI 复制未来牌袋。
- 使用系统时间、帧数或输入次数随机。
- `main.gd`、WorldBridge 或 OverlayLayer 中的影子牌架 owner。

历史存档迁移代码必须版本化、单向且完成后退出现役分支，不能静默继续影响新局。

## 13. 完成门槛

区域牌架随机化只有同时满足以下条件才完成：

1. 全部区域挂牌槽都来自统一合法池。
2. 工厂与市场没有默认先后关系。
3. 没有固定类别保证槽或类别刷新阶段。
4. 市场先出现、先购买、先建成是合法路径。
5. 只补购买产生的空槽。
6. gameplay RNG 可重放且与 UI 输入隔离。
7. 牌位、牌袋、唯一牌、刷新序号和事务 lineage 可精确保存恢复。
8. AI 只能读取公开牌位。
9. 区域购牌窗口是按需临时窗口。
10. 牌架 owner 仍然唯一，未向 `main.gd`、UI、AI 或 WorldBridge 扩散。
