## Hard product gates

- The real game must start, save, exit, restart, resume, finish, settle, return
  to menu, start again and close cleanly.
- Gameplay and AI use typed requests, stable targets and authoritative receipts.
- Cash, GDP, Sale Receipts, cards, monsters and wagers never duplicate.
- Rival private cash, hands, storage, ownership, annotations and AI plans never
  enter public or rival-scoped projections.
- Main gains no new caller or fallback; remaining core routes leave Main
  monotonically without creating another monolith.
- Alpha 0.3 content is data-only. A missing approved mechanic is a blocker, not
  permission to invent one.
- Fallback art remains labelled fallback. Release assets require provenance.

## Parallel ownership

Each cycle may carry one Product, one P0/P1 Architecture, one Save, one Content,
one Art, one UX/Audio and one Test-Debt task when their file sets are disjoint.
Shared composition, release, manifest and program-state files are integration
writer only. Feature branches request those edits through
docs/integration_requests/<task_id>.json.
