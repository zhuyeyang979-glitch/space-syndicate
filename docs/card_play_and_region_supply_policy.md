# 卡牌出牌与区域牌架策略

本文定义非商品普通牌的出牌资格、区域挂牌随机性、报价生命周期和公开信息边界。顶部免费商品履带是独立系统，不进入区域牌池。

## 出牌资格

- 默认门槛字段为 `play_requirement_kind = region_gdp_share`。
- 地区范围由 `play_region_scope` 指定：`target_region`、`contract_source_region` 或 `own_best_region`。
- 百分比由 `play_region_gdp_share_required` 指定。
- 普通 I-IV 梯度为 `0 / 15 / 25 / 35`。
- 高影响 I-IV 梯度为 `10 / 20 / 30 / 40`。
- `city_development`、`monster_bound_action`、`military_command` 始终免 GDP 门槛；它们分别受目标合法性、冷却或单位状态约束。
- `play_product` 只作为旧数据的显式迁移来源，现役规则字段为 `supply_product`，不构成默认出牌门槛。
- 现金购买费、资产出牌费和 GDP 资格互相独立。

GDP 份额按“该玩家在区域内获得的商品 GDP / 区域商品 GDP 总量”计算，并以 basis points 比较，避免临界百分比浮点误差。公开 UI 只显示牌面阈值和当前玩家是否合法，不显示对手真实份额或 AI 内部评分。

挂牌资格与当前玩家能否立即出牌是两件事。当前现金不足、资产不足、尚未拥有设施、当前目标被占用或当前 GDP 门槛不足，只影响购买/出牌提示，不会把该牌从随机牌池永久移除。

## 统一合法区域牌池

每个区域的全部挂牌槽位都从同一个合法非商品普通牌池抽取。不存在工厂阶段、市场阶段、类别解锁链或任何类别保证槽。

满足当前模式启用条件时，下列家族都可以出现在任意随机槽位：

- 工厂与市场设施牌。
- 道路、码头与空港牌。
- 六色仓库牌。
- 城市发展与其他策略牌。
- 怪兽与军队单位牌。
- 组织牌。
- 玩家互动与反制牌。

因此市场牌可以先于工厂牌出现，工厂牌也可以先于市场牌出现；区域暂时只挂牌交通、仓库、战术或单位牌同样是正常开局。不同区域可以得到完全不同的牌型组合。

合法池只允许以下硬过滤：

1. 卡牌在当前模式中启用。
2. 卡牌未退役。
3. 卡牌资源可加载，必需规则字段有效。
4. 全局唯一牌没有违反唯一性或已消耗状态。
5. 本局地图、席位数、模式能力与目标类型中至少存在一个潜在合法目标。

“潜在合法目标”判断永久或模式级可能性，不读取某位玩家当前现金、资产、设施、手牌、GDP、镜头、窗口是否打开或 AI 计划。只有本局规则上永远不可能生效的牌才被过滤。

需要权重时，只读取显式、可审计的 `region_supply_weight`。默认所有合法家族使用相同权重；权重只影响确定性抽取概率，不建立工厂优先、市场后置或其他隐藏顺序。内部权重不向玩家显示。

## 权威随机与牌袋

现役区域挂牌 owner 独占区域牌位、每区牌袋、刷新序号与全局唯一牌状态。随机只使用权威 gameplay RNG：

- 同一对局种子、同一合法池和同一购买序列必须产生相同挂牌序列。
- 不同种子应产生不同序列。
- 系统时间、帧数、鼠标位置、悬停、滚动、焦点变化、窗口打开次数和 AI 思考时长都不能成为随机源。
- UI、AI、WorldBridge 和 `main.gd` 不得复制牌袋或自行抽牌。

每个区域使用确定性 shuffle bag 或语义等价的无放回抽取。牌袋只在权威初始化、牌袋耗尽后的确定性重建，或合法池发生明确规则修订时改变；普通 UI 生命周期不能重建牌袋。

带权牌袋必须从稳定排序的合法条目生成，再由 gameplay RNG 确定顺序。相同权重的条目按稳定 `card_id` 作为随机前的规范顺序；抽取后不得再按类别重排。全局唯一牌抽出或消耗时立即更新唯一状态，其他区域后续补槽不得再次得到该牌。

## 槽位与购买生命周期

- 新局初始化时，按稳定区域 id 顺序为每个区域的模式定义槽位逐一抽牌。
- 购买成功只清空被购买的槽位，并以一次权威抽取补充该槽。
- 其他槽位保持原卡牌、报价关系和刷新序号，不重抽整个区域。
- 购买失败、报价过期、现金变化、目标变化和窗口关闭不会补槽。
- 打开、关闭、悬停、滚动、切换页签或重新打开区域牌架不会刷新。
- 多次收到同一购买结果时，exact-once transaction lineage 阻止重复扣款、重复发牌或重复补槽。

全部挂牌公开来源区域、卡牌身份、基础价和牌面条件。来源区域位于日照半球时才可购买，暗面只可浏览。公开预览不创建报价；玩家明确选择或确认某张挂牌后才建立 5 秒 `world_effective` 权威报价。

报价锁定挂牌身份、来源区域、日照资格、怪兽计数、价格倍率与最终价格。关闭窗口不延长报价；再次打开时仍有效则显示剩余时间，已过期则重新报价。购买成功默认关闭窗口；本地“连续浏览”设置可以保持打开，但不改变任何权威状态。

## 存档与读档

存档必须恢复：

- 每个区域每个槽位的当前 `card_id`。
- 每区牌袋剩余顺序，或可完全重现该顺序的 gameplay RNG 状态与抽取游标。
- 全局唯一牌的未抽取、已挂牌、已购买、已消耗等状态。
- 每区与全局刷新序号。
- 与购买和补槽关联的 exact-once transaction lineage。

读档后不得重新洗牌、重新填满已有牌位、跳过下一张牌或重复补槽。存档前后，在相同后续购买序列下必须继续得到相同牌序。

旧存档若缺少新牌袋字段，只允许通过带版本号的一次性迁移生成现役状态；迁移结果必须立即保存，后续读档不能再次迁移或改变牌序。

## 公开信息、AI 与隐私

玩家和 AI 都只能读取已经公开的区域牌位、公开牌面条件、公开日照与公开报价结果。AI 不得读取：

- 牌袋未来顺序。
- gameplay RNG 内部状态。
- 未公开唯一牌的位置。
- 其他玩家现金、手牌、资产、私人计划或 AI 评分。

AI 可以根据当前公开牌架评估购买、建设市场或工厂、连接积压市场以及释放仓库库存的价值。市场先出现时，AI 必须把它视为可以先建设并安装具体商品需求的准备型策略，不得假设必须先有工厂。

## 临时区域牌架窗口

区域牌架默认关闭且不占主桌布局空间。单击区域只改变区域选择；只有双击区域、选择“查看牌架”或对应键盘/手柄操作才打开牌架。

打开窗口不暂停世界，也不暂停日照、自转、报价或自动经济。Back、Esc 与手柄取消关闭最上层可关闭窗口并恢复打开前焦点。关闭牌架不会取消仍在有效期内的权威报价，也不会延长其寿命。

牌序竞价、怪兽下注、反制与目标选择不与区域牌架常驻堆叠。完整牌序竞价只在 `public_bid` 阶段出现；怪兽下注只在战斗强制决策出现。多个强制决策由 `ForcedDecisionRuntimeScheduler` 决定唯一当前表面，Presentation 不拥有计时、现金、报价、出价或牌队列。

## Owner 边界

- 现役区域牌架 / CardFlow / DistrictPurchase owner：唯一拥有合法牌池快照、区域牌位、确定性牌袋、刷新序号、唯一牌状态、报价和购买事务。
- `ForcedDecisionRuntimeScheduler`：只拥有强制决策优先级和阻塞语义，不拥有区域牌架或竞价经济状态。
- `OverlayLayer`：只呈现区域牌架和临时决策窗口，不拥有随机、报价、现金、卡牌或计时。
- AI：只提交基于公开牌位的意图，不抽牌、不刷新牌架。
- WorldBridge：只提供纯数据世界事实并原子应用权威 owner 已规划的结果，不判断下一张牌类别。
- UI 与 `main.gd`：不得保存影子牌袋、打开即刷新、类别阶段或兼容保证槽。

## 验收不变量

- 相同种子与购买序列得到相同挂牌序列；不同种子得到不同序列。
- 市场牌可以先于工厂牌，工厂牌也可以先于市场牌。
- 任意槽位都没有工厂、市场、城市发展或怪兽保证语义。
- 购买一槽只补一槽；其他槽位身份不变。
- UI 打开、关闭、悬停、滚动与重新打开不改变牌位或牌袋。
- 读档后牌位不变，后续抽取顺序继续一致。
- 当前买不起或暂时打不出的牌仍可挂牌并显示条件。
- AI 和公开快照不包含未来牌袋、RNG 状态或私人玩家信息。
- 区域牌架关闭后不占布局；单击区域不自动打开。
- 完整牌序竞价只在 `public_bid` 阶段出现，怪兽下注只在战斗决策出现。
- 玩家文本不出现权重、RNG 状态、owner、snapshot、raw state enum 或 AI 私有评分。
