# Planet Table Layout Captures

`RoleTablePortraitPreviewCapture.tscn` writes 3, 4, 6, and 8-seat Godot runtime
captures here. This is a **QA-only composition study**. Its `PlanetSeatLayout`
algorithm is not a production seat owner, is not connected to `PlanetBoard`,
and must never be used as evidence of an in-place eight-seat upgrade.

These captures validate portrait readability, inward-facing direction and
privacy presentation only. They do not validate production player-to-seat
mapping, current planet input, or completion of all third-party portraits.
