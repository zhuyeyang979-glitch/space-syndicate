# 《太空辛迪加》v0.6 卡牌效果确认清单

日期：2026-07-14

本清单只确认卡牌效果，不改胜利、经济或地图基础规则。现金购买费和非商品牌资产费用目前用于跑通流程，不视为最终平衡值。

## 1. 商品牌：184 张，核心效果 adapter 已验证

- 46 个商品家族，每个 I–IV 级。
- 免费从顶部履带领取；免费打出。
- 安装到同色工厂或市场，分别永久增加生产率或需求率。
- 等级增量固定为 `10 / 20 / 40 / 80` 单位/分钟。
- 受设施容量、区域共享生命效率、真实路线、仓库和市场处理能力限制。
- 商品归打出者；使用他人设施时按公共设施规则付租。

待补内容不是卡牌效果，而是六色商品数量补齐：还缺 20 个商品家族的名称与商品目录记录。

当前独立集成测试已经通过真实 `CommodityFlowRuntimeController.install_commodity()` 完成安装、幂等重放、设施方向与同色校验；接入主场景前仍需通过统一玩家状态端口取得手牌和资产真相。

建议锁定：升级时同时恢复本次增加的最大生命差值；同级或低级牌用于修复时恢复该牌等级的生命贡献，但不超过区域最大共享生命。租金以真实通过量/生产量/销售量/仓储量计费，并受成交价值总租金上限约束。

当前独立集成测试已经通过真实 `RegionInfrastructureRuntimeController.apply_facility_action()` 完成设施建造、唯一槽位、产权、等级、仓库产业选择与 region revision 校验；不会覆盖生产 owner。

## 3. 供需牌：8 张，权益 planner 已验证，生产 batch sink 待接

- 远洋采购令：产生一次性额外需求；海运标签；距离大于 2。
- 近地供货潮：产生一次性实体供货；陆运标签；距离不超过 2。
- 等级数量固定为 `20 / 40 / 80 / 160`。
- 打出者必须拥有符合条件商品的正 GDP。
- 多式联运只要路线含指定运输方式即满足标签，整条路线只结算一次。
- 仍需真实工厂、市场、路线、吞吐和仓储；只在最终成交后产生现金、GDP 与资产。
- 收益完全跟随商品 GDP，不额外奖励原始工厂所有者。

建议把相邻免建路交易标记为“本地直达”，不自动获得“陆运”标签，避免没有道路却触发陆运条件。

已实现并验证：按“商品所有者 + 具体商品”的最近 30 秒 GDP 聚合；纯整数最大余数；容量不足后的迭代再分配；共享运输资源不超卖；使用 `shortest_legal_distance` 和 `topology_revision`；`local/direct` 不伪装成陆运；多式联运任一段满足标签即合法。

生产阻塞：现有商品流只有单次供货原语，没有订单 + 供货统一的原子 batch/rollback API。当前 adapter 强制要求注入 batch sink；缺失时失败闭合，卡牌和资产不消耗，不会伪造商品、成交、GDP、租金或资产。

## 4. 怪兽牌：32 张，部署规则已定，动作表待定

已定：没有同名怪兽时部署；已有同名怪兽时升级并恢复满血，剩余在场时间增加 60 秒且不重置；IV 级重复打出仍恢复满血并增加 60 秒。

建议锁定：高级牌直接把同名怪兽提升至 `max(当前等级, 牌面等级)`；同级牌只治疗和延时。仍需给八只怪兽分别定义基础生命、移动、视野/自动开战半径、单位伤害、区域共享生命伤害、绑定动作资产费用和冷却。

## 5. 军队牌：28 张，单位定位已定，动作数值待定

七个家族为行星防卫军、制空战斗机、轨道轰炸机、重装坦克、导弹阵地、潜航舰队和星海战舰。绑定动作不占五张普通手牌，但执行时消耗资产并进入冷却。

必须把 `unit_combat_damage` 与 `region_shared_hp_damage` 分开；旧版 1–5 点伤害不能直接用于 100–400/设施的共享生命模型。建议达到控制上限后由玩家选择替换一个己方单位，不自动替换最旧单位。

## 6. 玩家互动牌：12 张，4 张已定、8 张待简化

- 相位否决 I–IV：5 秒响应窗口内否决一张玩家互动牌；首版只允许一层反制。核心语义可接入。
- 星链拆解 I–IV：当前暂定随机弃牌、短时锁定，并在高等级附加现金惩罚。建议删除现金惩罚，只保留弃牌和短锁定。
- 影仓牵引 I–IV：当前暂定偷牌、短时锁定和失败补偿。建议删除失败补偿；目标满手且不能立即合法合成时，本次效果无效，不销毁牌、不凭空发钱。

## 7. 资产费用说明

- 现金只用于购买非商品牌。
- 商品牌领取与打出均免费。
- 非商品牌打出支付对应颜色资产。
- 无色牌支付通用资产；玩家可指定六色资产的组合，未指定时系统按可用余额自动分配。
- 不存在第七个“通用资产池”。

## 建议的确认顺序

1. 先接通商品与设施，验证完整生产—运输—仓储—销售循环。
2. 再接通供需牌，验证全局一次性供需不会绕过物流。
3. 再锁定怪兽/军队动作表和区域伤害梯度。
4. 最后简化并锁定星链拆解、影仓牵引和相位否决的等级差异。
